Publication Strategy

Как оформить GitHub правильно

1. GitHub профиль

Сделайте pinned-репозитории: `shanyraq`, `print-sales-assistant-bot`, `fond4180-crypto.github.io` и при возможности ещё один маленький AI/CV кейс.

2. Отдельный GitHub Pages

Лучший вариант: отдельный репозиторий `username.github.io`, где будет одна понятная страница с проектами, достижениями и контактами.

3. README каждого проекта

У каждого публичного репозитория должен быть понятный README: задача, стек, скриншоты, архитектура и ваш вклад.

4. Figma

Если есть сильные дизайны в Figma, добавьте ссылку или 2-3 превью как proof of product thinking, особенно для `Shanyraq`. Если пока не хотите светить ссылки, можно просто указать, что product/UI designs are available on request.
